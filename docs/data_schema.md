# Data Schema

## Kvasir-VQA-x1

Expected location:

```text
step3_dataset_migration/raw/kvasir_vqa_x1/
├── train.parquet
└── test.parquet
```

Expected core columns:

| Column | Meaning |
|---|---|
| `img_id` | Image identifier used to match images and questions. |
| `question` | Natural-language visual question. |
| `answer` | Ground-truth benchmark answer. |
| `complexity` | Dataset-provided complexity label used for routing buckets and analysis. |
| `question_class` | One or more semantic question classes. The first class is treated as the primary class. |

Optional columns depend on the image download path:

| Column | Meaning |
|---|---|
| `image_url` | Remote image URL used by manifest builders and downloaders. |
| `image_path` | Local relative image path used by server bundles. |

## Qwen Cache JSONL

The full Qwen cache is regenerated by `run_qwen_sanity.py`. Each row should contain:

```json
{
  "id": "test-000000",
  "img_id": "image-id",
  "question": "What abnormality is present?",
  "answer": "polyp",
  "complexity": 1,
  "question_class": ["abnormality_presence"],
  "qwen_direct": "polyp",
  "qwen_evidence_raw": "{\"lesion_presence\":\"yes\"}",
  "qwen_evidence": {
    "lesion_presence": "yes",
    "lesion_type": "polyp",
    "lesion_count": "1",
    "location": "sigmoid colon",
    "instrument_presence": "no",
    "text_overlay_presence": "no",
    "abnormality_presence": "yes",
    "uncertainty": "low",
    "evidence_sentence": "A polyp-like lesion is visible."
  }
}
```

Some scripts use `parsed_evidence` instead of `qwen_evidence`; the released code contains compatibility logic for common variants.

## Template Prediction CSV

The cache-only scripts expect a generated prediction CSV with columns such as:

| Column | Meaning |
|---|---|
| `id` | Sample identifier. |
| `answer` | Ground-truth answer. |
| `tfidf_direct_template` | Direct-template retrieval answer. |
| `tfidf_evidence_rerank` | Evidence-aware reranked answer. |
| `question` | Question text. |
| `complexity` | Complexity label. |
| `question_class` | Question class list or primary class. |
| score columns | Optional retrieval/reranking scores used by calibration features. |

## SLAKE

Expected location:

```text
data/slake/
```

The SLAKE script is written to tolerate common SLAKE JSON naming patterns. Use:

```bash
python step5_experiments/server_scripts/slake_external_validation.py inspect --data-dir data/slake
```

to verify the detected splits before running inference or evaluation.
